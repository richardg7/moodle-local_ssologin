## [v2.0.0-dev] – 2026-05-09

### 🔒 Security & Architecture
- ✅ **Authenticate-then-Decrypt flow**: HMAC-SHA256 signature is now verified *before* any decryption attempt, preventing Padding Oracle attacks
- 🔁 **Nonce-based Replay Protection**: Each request may include a nonce; once consumed, subsequent use of the same nonce is rejected
- 🔐 **Clock-skew tolerance**: Bidirectional timestamp validation with configurable expiry window
- 🔗 **Secure Redirect**: After login, the `redirect` parameter is validated — only internal Moodle URLs (same host) are accepted; external hosts are silently ignored and redirected to `/`

### ⚙️ New Admin Settings
- 🏛️ **Legacy Mode** (`legacymode`): Toggle that allows requests missing an HMAC signature to still be processed — intended solely for transition periods while integrations are updated. Requests using this path are flagged via `debugging()` to alert administrators
- 👤 **JIT Provisioning** (`jitprovisioning`): Automatically creates a Moodle user account on first SSO login when the user does not yet exist. Requires `email`, `firstname`, and `lastname` in the payload
- 🔄 **Profile Sync** (`profilesync`): On every SSO login, overwrites Moodle user profile fields (`email`, `firstname`, `lastname`, `lang`, `country`, `city`, `institution`, `department`) with values from the payload

### 📦 New Classes & Database
- 📡 **Event API** (`classes/event/sso_login_attempted.php`): Moodle event class registered for every SSO login attempt (success or failure), visible in Moodle's native log reports
- 🗄️ **Database schema** (`db/install.xml`): New table `local_ssologin_nonces` with unique index on `nonce` column to ensure replay-attack protection at the database level

### 🛡️ Privacy (GDPR)
- 📜 **Privacy API** (`classes/privacy/provider.php`): Full implementation of `core_privacy\local\metadata\provider` and `core_privacy\local\request\plugin\provider`, declaring nonce storage and its purpose

### 📦 Code Quality
- All new functions documented with PHPDoc blocks
- Moodle Coding Standards applied throughout (`core_text`, `user_create_user`, `user_update_user`)
- Version bumped to `2026050900`

## [1.0.0] – 2025-04-20

### 🚀 Initial release

- 🔒 Full implementation of the SSO login plugin for Moodle
- 🔑 Support for authentication via shared secret key (HMAC)
- 🔐 AES-256-CBC encryption with random IV and Base64 encoding
- ⏱️ Timestamp validation with configurable tolerance
- ✅ HMAC-SHA256 signature validation
- 📥 Support for automatic authentication based on GET parameters
- 🧾 Authentication log generation (success/failure)
- ⚙️ Configuration screen in the Moodle admin panel
- 📚 Examples of external integration in PHP, Python and Java
